Models:

In LangChain, In LangChain, "models" are the core interfaces through which you interact with Al models.

LLMs have solved the problem of NLP and replying to the question with appropriate answer(context aware text generation)

LangChain standardizes the model creation and interaction


LangChain helps to interact with two types of models:
    Language Models: text in text out
    Embedding Models: text to vector output

Chains:
    we can build piplines

    types of chains:
        Extended: all the nodes are like sequential manner. Combines the prompt template, model invocation, output parsing, and additional processing steps into a single chain using LangChain Expression Language (LCEL)
            
        parallel chains: these are like having two things running parallel may be they can intersect at the end

        Branching chains: Have different branches just like we have in the trees
        


Indexes:
Indexes connect your application to external knowledge-such as PDFs, websites or databases
    Doc loader: to load the document
    text splitter: to make it into chunks
    Vector store: for storing the Embeddings
    Retrivers: generates Embedding and finds the relavant things using semantic search



Memory:
    LLM apis are stateless, doesn't have memory of the previous question

    Memory will solve this problem

    Types of memory:
        Conversation Buffer Memory: Stores a transcript of recent messages. Great for short chats but can grow large quickly.

        ConversationBuffer WindowMemory: Only keeps the last N interactions to avoid
        excessive token usage.

        Summarizer-Based Memory: Periodically summarizes older chat segments to keep
        a condensed memory footprint.

        Custom Memory: For advanced use cases, you can store specialized state (e.g., the user's preferences or key facts about them) in a custom memory class.
